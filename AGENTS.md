# AGENTS.md

## Review focus
- 可読性、責務分離、不要な複雑化を優先して指摘する
- 破壊的な変更や大量リネームは事前に確認する

## Validation
- 変更後はビルドエラー、テスト、主要な実行フローを確認する
- 推測で修正せず、根拠が弱い場合は確認事項として残す

## UI freeze mitigation notes
- `ICredentialStore.Load()` を UI スレッド前提にしない。背景化した呼び出しで壊れない設計にする
- `Task.Run` は強制停止できないため、Keychain / `security` が詰まる可能性とバックグラウンド残留を前提に扱う
- 非同期化でエラー表示の順序やタイミングが変わりうるため、ユーザー体験の変化を確認する
